🔊 How Alexa Integration Works

Since Belkin shut down the official WeMo Alexa skill, you can set up your own custom Alexa skill that talks to WeMoHub running on your local network. You need your server to be reachable from the internet (use Tailscale, ngrok, or port forwarding — you already have Tailscale set up!).

1 Expose WeMoHub to the Internet

Using your existing Tailscale setup, your WeMoHub URL is:

http://[your-tailscale-ip]:5000/alexa

Or use ngrok for a quick HTTPS tunnel:

ngrok http 5000

2 Create a Custom Alexa Skill

Go to developer.amazon.com → Alexa Developer Console → Create Skill
Choose "Custom" model and "Provision your own" backend
Under "Endpoint", paste your WeMoHub URL above

3 Add These Intents to Your Skill

Create these intents in the Alexa console (Interaction Model → Intents):

TurnOnIntent
Sample utterances:
  "turn on {DeviceName}"
  "switch on {DeviceName}"

TurnOffIntent
Sample utterances:
  "turn off {DeviceName}"
  "switch off {DeviceName}"

ToggleIntent
Sample utterances:
  "toggle {DeviceName}"
  "flip {DeviceName}"

Add a slot DeviceName of type AMAZON.DeviceName or a custom slot with your device names.

4 Use Your Device Names

Name your devices in the Devices tab using aliases that are easy to say. Then just say:

"Alexa, ask WeMo to turn on living room"
"Alexa, ask WeMo to turn off office plug"
"Alexa, ask WeMo to toggle garage switch"

💡 Simpler Alternative: Home Assistant + Nabu Casa

If you run Home Assistant on your network (e.g. on the same machine or a Pi), it has a built-in WeMo integration via pywemo and native Alexa support through Nabu Casa (~$7/month). This gives you zero-config Alexa voice control, a full dashboard, and advanced automations without any custom skill development.
